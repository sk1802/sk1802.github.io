Core Metrics

Every repository is scored on four dimensions that describe its technical characteristics and evolution pattern.

🔨 Craftsmanship (0-100)
Reflects code quality, documentation, and maintainability. High scores indicate mature projects with strong engineering practices, comprehensive tests, and active code review. Low scores suggest rapid iteration or incomplete documentation.

Signals: Code review depth, documentation completeness, test coverage, commit message quality, consistent style.
⚙️ Machine Influence (0-100)
Measures how much the codebase appears generated, templated, or auto-scaffolded. High scores indicate projects with repetitive patterns, generated boilerplate, or heavy use of code generation tools. Low scores suggest hand-written, artisanal code.

Signals: Example density, template directories, generated code patterns, boilerplate ratio, tool-generated structure.
✓ Stability (0-100)
Indicates maintenance continuity and release predictability. High scores show consistent commit activity, regular releases, and predictable governance. Low scores suggest irregular or sporadic maintenance.

Signals: Commit frequency consistency, release regularity, issue resolution time, maintainer continuity.
🚀 Velocity (0-100)
Reflects how quickly the project moves: commit frequency, release rate, and feature addition pace. High velocity means rapid development. Low velocity suggests slower, more deliberate evolution.

Signals: Commits per month, release frequency, pull request throughput, issue closure rate.

Civilizations

Five patterns describe how repositories evolve and move through development cycles. A civilization reflects not quality, but development philosophy and maturity stage.

Vector Sigma
Ancient craftsmanship. Slow, deliberate systems with long institutional memory. High stability, moderate velocity. Examples: Linux kernel, PostgreSQL, OpenSSL.
Autobots
Humans augmented by automation. High craft with visible acceleration. Strong tooling and CI/CD integration. Examples: Kubernetes, TypeScript, React.
Decepticons
Fast-moving frontier projects where velocity outpaces refinement. High activity, newer, sometimes chaotic. Examples: Startup frameworks, experimental tools.
Quintessons
Factory worlds and generated surfaces. Highly automated, highly repeatable. High machine influence. Examples: Build tools, code generators, scaffolders.
Drifters
Unclassified frontier projects with incomplete signal. Experimental, emerging, or low-activity repositories. Too new or uncertain to categorize confidently.

Key Concepts

Technical DNA: The semantic and structural fingerprint of a repository. Includes language profile, dependency patterns, code organization, and readme signals. Used to position projects in the 2D atlas space.

Semantic Vector: A numerical representation of what a project does based on its description, topics, code patterns, and documentation. Similar vectors cluster nearby in the atlas.

Dependency Graph: The network of libraries, frameworks, and packages a repository relies on. Projects with similar dependencies tend to cluster together.

Craftsmanship Score: A heuristic combining code review depth, test coverage, documentation quality, and commit message coherence. Not a judgment of worth, but a signal of maintenance rigor.

Machine Influence Score: Detects generated or templated code patterns. High scores indicate boilerplate, scaffolding, or code generation. Used to identify factory-style projects.

Cluster: A neighborhood of related repositories grouped by technical similarity. Clusters have names (e.g., "Kernel Systems") and visual boundaries on the map. Each cluster represents a distinct technical domain.

Civilization: A development philosophy pattern across 5 types: Sigma (ancient), Autobots (augmented), Decepticons (fast), Quintessons (generated), Drifters (unclassified). Describes evolution style, not quality.

Nearest Neighbors: The 5 closest repositories in technical space. Not based on shared stars or follows, but on structural and semantic similarity. Shows your technical peers.

Confidence Score: A trust metric (0-100%) for how confident the atlas is about a repository's cluster assignment. Based on signal consistency and metric coherence.

Wrapping Map: The atlas edges wrap horizontally, creating a toroidal topology. You can scroll infinitely left or right and come back to where you started. This allows the entire ecosystem to be viewable at once.

Static Data: All data is pre-computed and stored as JSON files. No API calls at runtime. This makes the atlas fast and works perfectly on GitHub Pages (no server needed).
